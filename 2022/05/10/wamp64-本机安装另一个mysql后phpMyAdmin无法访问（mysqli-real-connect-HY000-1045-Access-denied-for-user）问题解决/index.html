wamp64 本机安装另一个mysql后phpMyAdmin无法访问（mysqli::real_connect(): (HY000/1045):Access denied for user）问题解决

gala-2nd

2022-05-10 23:54:18

前言

今天回看一个月前写的php项目，发现mysql服务无法启动，想起最近有mysql课程又添加了一个8.0的mysql，因此判断两者端口冲突。遂决定更改wamp64中mysql的端口。

图中提示
mysqli::real_connect(): (HY000/1045): Access denied for user ‘root‘@’localhost’ (using password: NO)


# 一、在wamp64菜单中更改
1.首先右键wamp64图标，选择Tools,然后在port used by MySQL一项选择use a port other than xxx （这里已经被更改过为3308）；或是左键选择wamp64图标，选择mysql，然后在port used by MySQL一项选择use a port other than xxx （这第二种方法可能不会显示）

2.然后会跳出来一个输入框，填入3301到3309之间的端口号，（注意mariaDB已经使用了3307，因此不能使用3307。）

3.点击ok后wamp64会自己重启服务，启动完成后就可以登录phpMyAdmin了。

二、在文件中更改

1.左键wamp64图标，选择mysql，再选择my.ini打开

2.在my.ini中查找3306，（3306为mysql连接数据库默认端口）更改为3301到3309之间的端口号，注意mariaDB已经使用了3307，因此不能使用3307。（一般会查找到三个没有被注释掉的3306，将这三个更改掉即可,不同的编辑器查找方式会不同，可以试试ctrl+f）

3.在wamp64安装目录下找到wamp64manager.conf打开

4.在wamp64.conf里面找到mysqlPortUsed，更改为第2步中在my.ini更改的端口号

5.保存后重启wamp服务，phpMyAdmin就可以访问了

总结

端口号冲突经常出现在wamp64使用的过程中，当出现异常时，可以考虑对相应的服务更改端口号。
